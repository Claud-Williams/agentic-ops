# Estimate log (example)

Fictional persona: **Jordan Vale**. Predicted versus actual effort, grouped by task type.

| Task | Type | Predicted | Actual | Variance |
|---|---|---|---|---|
| Draft the depot rollout brief | analytical write-up | 60 min | 18 min | ~3.3x faster |
| Reconcile two depot rotas | data clean-up | 30 min | 75 min | ~2.5x slower |
| Write the rollout checklist | checklist | 20 min | 15 min | about right |

## The bands that emerge

- **Analytical write-ups** run consistently three to six times faster than they feel, so estimate low.
- **Data clean-ups** routinely run over, so estimate high.

Before the next estimate of either type, read the band and apply it rather than guessing fresh. That single read-back step is what stops the same bias repeating.
